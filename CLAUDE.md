# Stock Insight Project

## Instructions

You must read all files in /docs before coding.

Follow the architecture and feature specifications strictly.

## Development Order

1. Database schema
2. Data ingestion script
3. API routes
4. Analysis logic
5. Frontend pages

## Rules

- Use TypeScript
- Use Next.js App Router
- Keep code modular
- Separate logic and UI

## 커밋 규칙

커밋은 작업 단위별로 나눠서 한다.

### 타입

| 타입 | 설명 |
|------|------|
| `feat` | 새 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 동작 변경 없는 코드 개선 |
| `style` | UI/스타일 변경 |
| `chore` | 설정, 의존성, 문서 등 |

### 형식

```
타입: 한글로 짧게 설명

- 변경된 파일 또는 주요 내용을 bullet로 나열

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

## Important

This project uses T+1 stock data.

Do NOT implement real-time features.

Focus on:

- data analysis
- beginner-friendly explanations
