# Task 05: 프론트엔드 페이지 구현

## 목표

Next.js App Router 기반으로 Stock Insight 서비스의 UI 페이지를 구현한다.
초보자 친화적인 디자인과 설명을 중심으로 작성한다.

## 사전 확인 사항

- Task 03, 04 완료 후 진행 (API가 정상 동작해야 함)
- `docs/feature-spec.md` 의 기능 명세를 기준으로 UI 구성
- Tailwind CSS 사용, Recharts로 차트 구현
- 모든 페이지는 서버 컴포넌트 기반, 클라이언트 컴포넌트는 최소화

## 구현할 페이지 및 컴포넌트

### 페이지

```
src/app/
├── page.tsx                  # 홈 (시장 리포트)
├── ranking/
│   └── page.tsx              # TOP 랭킹
└── stock/
    └── [code]/
        └── page.tsx          # 종목 상세
```

### 공통 컴포넌트

```
src/components/
├── Header.tsx                # 상단 네비게이션 + 검색창
├── StockChart.tsx            # 종가 라인 차트 + MA 오버레이 (클라이언트)
├── RankingTable.tsx          # 랭킹 테이블
├── MarketSummary.tsx         # 시장 요약 카드
├── AnalysisCard.tsx          # 분석 지표 카드 (MA, 변동성, 확률 등)
└── GuideTooltip.tsx          # 초보자 설명 툴팁
```

## 각 페이지 구현 지침

### 홈 페이지 (`/`)

`/api/market` 데이터를 가져와 표시한다.

구성 요소:
- 상단: 서비스 소개 한 줄 문구 ("어제 데이터로 오늘을 공부해요")
- 기준일자 표시 ("2025년 3월 25일 기준")
- 시장 요약 카드 (상승 종목 수 / 하락 종목 수 / 평균 등락률)
- 상승률 TOP 3 / 하락률 TOP 3 / 거래량 TOP 3 미리보기
- 각 TOP 항목은 클릭 시 종목 상세로 이동
- "전체 랭킹 보기" 링크 → `/ranking`

### 랭킹 페이지 (`/ranking`)

`/api/ranking?type=rise` 데이터를 가져와 표시한다.

구성 요소:
- 탭: 상승률 / 하락률 / 거래량 / 거래대금 (탭 전환 시 URL 쿼리 파라미터 변경)
- 랭킹 테이블: 순위, 종목명, 종목코드, 현재가, 등락률, 거래량
- 종목명 클릭 시 `/stock/[code]` 로 이동
- 탭은 클라이언트 컴포넌트로 구현 (`'use client'`)

### 종목 상세 페이지 (`/stock/[code]`)

`/api/stocks/[code]` 와 `/api/analysis/[code]` 데이터를 가져와 표시한다.

구성 요소 (위에서 아래 순서):

1. **종목 기본 정보**
   - 종목명 + 종목코드
   - 종가, 등락률 (색상: 상승=빨강, 하락=파랑)
   - 시가 / 고가 / 저가 / 거래량 / 거래대금

2. **가격 차트** (StockChart 컴포넌트)
   - 최근 30일 종가 라인 차트
   - MA5, MA20 오버레이 (토글 가능)
   - x축: 날짜, y축: 가격

3. **분석 지표 카드** (AnalysisCard 컴포넌트)
   - 이동평균 (MA5, MA20, MA60) - 현재가와 비교
   - 변동성 등급 (낮음 / 보통 / 높음)
   - 최근 30일 상승 확률 (%)
   - 추세 판단 결과
   - 각 지표 옆에 GuideTooltip으로 초보자 설명 표시

4. **데이터 기반 예측 섹션**
   - 상승 확률 시각화 (프로그레스바)
   - 추세 신호 (아이콘 + 텍스트)
   - 면책 문구: "이 정보는 과거 데이터 기반 분석이며 투자 권유가 아닙니다"

5. **비슷한 종목** (상관계수 높은 종목 3개)
   - 종목명 + 상관계수 표시
   - 클릭 시 해당 종목 상세로 이동

## Header 컴포넌트 (공통)

- 로고 ("Stock Insight") + 홈 링크
- 종목 검색창: 입력 후 엔터 또는 버튼 클릭 시 `/stock/[code]` 또는 검색 결과로 이동
- `/api/stocks?q=검색어` 호출하여 자동완성 (클라이언트 컴포넌트)
- 네비게이션: 홈 / 랭킹

---

## 디자인 원칙

- 모바일 우선 반응형 (sm / md / lg 브레이크포인트)
- 색상: 상승=red-500, 하락=blue-500, 보합=gray-500 (한국 주식 관례)
- 초보자를 위한 설명은 작은 글씨로 카드 하단에 배치
- 로딩 상태는 Tailwind의 `animate-pulse` 스켈레톤 사용
- 에러 상태는 간단한 메시지로 표시 ("데이터를 불러오지 못했습니다")
